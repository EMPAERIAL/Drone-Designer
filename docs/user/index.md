# User Documentation

This section is for people running Drone Designer to produce UAV designs from mission requirements. It focuses on operator tasks, expected workflow, and common troubleshooting paths rather than internal implementation details.

## Start With Installation

- [Installation And Prerequisites](installation-and-prerequisites.md): how to prepare and launch the packaged application.

## Core Workflows

- [Design A UAV From Requirements](workflow-design-a-uav-from-requirements.md): enter mission inputs, run the design flow, review the result, and iterate.
- [Export And Generate Outputs](workflow-export-and-generate-outputs.md): export reports and move into downstream output-generation steps.

## If Something Goes Wrong

- [Troubleshooting And FAQ](troubleshooting-and-faq.md): operator-facing failure modes, checks, and recovery guidance.

## Reading Order

For most users, the intended path is:

1. Install and launch the app.
2. Design a UAV from requirements.
3. Export outputs if needed.
4. Use troubleshooting only when the normal workflow breaks down.
